# Pràctica mkdocs-grup3  
  
## Passos a seguir:  
Un dels integrants ha preparat un projecte *markdown*  des de la seva màquina virtual i l'ha preparada per pujar-la a *GitHub*. La resta d'integrants han fet un *git  clone* per agafar el repositori i tenir-la a cada màquina virtual.  
El treball s'ha repartit de manera que cadascú fa els fitxers corresponents de tal manera que la resta d'integrants no modifican el que no és seu per evitar problemes amb les pujades d'arxius.  Tothom treballa dins de la carpeta docs, que és on s'enmagatzemen tots els arxius *.md*. També s'ha modificat l'arxiu *.yml* per personalitzar la barra de navegació.
  
Per pujar els canvis a *GitHub* s'han seguit les següents ordres:  
 - *git  add* per afegir els elements modificats  
 -    *git  commit -m "missatge"* per deixar un comentari de que s'ha modificat  
 -    *git  pull* per pujar els canvis al repositori.

IMPORTANT: sempre que demani *password* de *GitHub* s'ha d'introduir el *token* que hem hagut de crear prèviament  des del nostre compte de *GitHub*.
